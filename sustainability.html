This page explains how the site is built, what files it depends on, and how a future user could revise or expand it.

Platform and Structure

The site is a static HTML and CSS project hosted on GitHub Pages. It does not require a server, database, content management system, or external JavaScript framework. This keeps the project transparent: the files in the repository are the same files that make the public site.

The public website presents generated HTML pages, while the analysis itself is preserved in the data tables and Python scripts that produced the frequency and KWIC results.

Essential Files

  • index.html - project overview, research question, and corpus description
  • findings.html - subject-level comparison and representative KWIC examples
  • subjects/ - one page for each school subject
  • books/ - one page for each textbook
  • planning.html - design logic and planning sketches
  • sustainability.html - platform and maintenance documentation
  • style.css - shared visual styling across the site
  • data/ - CSV files used by the website
  • images/ - textbook covers and planning sketches

Data Sources

The current corpus contains twenty-five textbooks from the 1930s: five books each from history, civics, geography, language readers, and self-cultivation. The corpus can be expanded by adding more cleaned text files and rerunning the analysis script.

The public site does not need the full raw extraction workflow to function. It depends on the cleaned tables already generated from the corpus, especially the subject frequency table, book frequency table, and KWIC table.

Future Use and Maintenance

Updating the site mainly means revising text, adding or replacing images, regenerating the HTML pages from the current CSV files, and checking that links still work. Because the site is static, it can also be moved from GitHub Pages to another static hosting platform without changing the basic structure.
