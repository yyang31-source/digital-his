Sustainability

This page explains how the site is built, what files it depends on, and what a future user would need in order to revise or maintain it.

Platform and Structure

This site is a static HTML and CSS project hosted on GitHub Pages. It does not use a content management system, server-side processing, or external JavaScript frameworks. Each page is a plain HTML file linked through a shared navigation bar and styled through a single stylesheet.

This platform was chosen because it is lightweight, transparent, and easy to maintain. The same files stored in the GitHub repository are the files that generate the public site.

Essential Files

  • planning.html — sketches and design decisions

Source Base

The site is based on a comparison of two Chinese textbooks from the early twentieth century, with particular attention to changing descriptions of the Yellow Emperor. The findings page also includes a comparison with a passage from the Shiji in order to show how later textbook narratives reused and reframed earlier historical language.

Because this site is designed as a teaching prototype rather than a full digital archive, it presents selected translated excerpts rather than full transcriptions or a complete corpus.

Future Use and Maintenance

A future user would need access to the GitHub repository and basic familiarity with HTML and CSS. Updating the site would mainly involve revising page text, adding or replacing images, and extending the findings page with additional textbook examples.

The project could be expanded by adding page images from the original textbooks, fuller bibliographic references, or more case studies showing how textbook narratives changed across time. If GitHub Pages became unavailable, the site could be moved to another static hosting platform without changing its basic structure.
